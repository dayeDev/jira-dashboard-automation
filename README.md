## 🎯 개요

### 왜 만들었나

QA 업무에서는 도메인마다 에픽을 만들어 이슈를 등재합니다. 문제는 진척률·우선순위·담당자별 현황을 파악하려면 **Jira 대시보드를 도메인마다 수동으로 만들어야 하고, 필터도 하나씩 따로 생성**해야 했다는 것입니다.

**반복되던 불편함들:**

- **자동화 부재** — 에픽 링크 하나면 충분한데, 대시보드와 필터를 매번 수동으로 생성
- **우선순위 확인 불가** — Jira 필터에서 우선순위가 보이지 않아 탭 여러 개 필요
- **해결됨 이슈 추적** — 재확인해야 할 티켓을 한눈에 파악하기 어려움
- **담당자별 현황 파악 불가** — 각 개발자의 상태를 알기 위해 여러 필터 필요
- **이슈 생성 반복 작업** — 보고자·레이블·에픽 링크 등 고정값을 매번 수동 입력
- **보고 자료 분산** — 협업 부서와 공유할 때 여러 링크를 따로 전달해야 함

## ✨ 주요 기능

| 기능 | 설명 |
|------|------|
| **에픽 자동 조회** | 에픽 키 하나로 해당 이슈를 모두 자동 로드 |
| **우선순위 탭** | 한 화면에서 우선순위별로 이슈 확인 |

## 🚀 시작하기

### 필수 요구사항

- **Node.js** 14.0 이상
- **npm** 6.0 이상
- Jira 계정 및 Personal Access Token(PAT)

### 설치

```bash
# 저장소 클론
git clone https://github.com/dayeDev/jira-dashboard.git
cd jira-dashboard

# 의존성 설치
npm install
```

### 실행

```bash
# 개발 서버 시작 (기본 포트: 3000)
npm start

# 프로덕션 빌드
npm run build
```

### 초기 설정

1. 브라우저에서 `http://localhost:3000` 접속
2. 헤더의 설정 버튼에서 Jira 정보 입력
   - **Jira 베이스 URL**: `https://your-domain.atlassian.net`
   - **Personal Access Token**: [Jira에서 발급](https://id.atlassian.com/manage/api-tokens)
3. 에픽 키 입력 (예: `PROJ-123`)
4. "불러오기" 버튼 클릭

## 🛠️ 기술 스택

### 백엔드
- **런타임**: Node.js
- **역할**: Jira REST API 프록시, CORS 우회
- **인증**: Personal Access Token(PAT)

### 프론트엔드
- **마크업**: HTML5
- **스타일**: 순수 CSS3 (외부 프레임워크 미사용)
- **로직**: Vanilla JavaScript (ES6+)
- **차트**: Chart.js 4.4.1

### 개발 도구
- **AI 페어 프로그래밍**: Claude Code
- **배포**: GitHub Pages

## 📖 사용 방법

### 1. 대시보드 조회

```
[에픽 키 입력] → [불러오기] → 대시보드 자동 생성
```

- **우선순위 탭**: 높음/중간/낮음/선택안함별로 이슈 필터
- **완료율 차트**: 전체·완료·잔여 이슈 시각화
- **담당자 드로어**: 각 개발자별 진행 상황 확인

### 2. 이슈 생성

```
[이슈생성 버튼] → 모달 팝업 → [필드 입력] → [만들기]
```

**자동 채워지는 필드:**
- 프로젝트
- 보고자
- 에픽 링크
- 레이블

**추가 입력 가능:**
- 요약, 설명, 담당자, 우선순위, 첨부파일, 연결 이슈

### 3. 스냅샷 내보내기

```
[내보내기] → 현재 대시보드 상태가 HTML 파일로 저장 → 협업 부서에 공유
```

저장된 파일은 **Jira 연동 없이도 열 수 있는 읽기 전용 뷰**입니다.

## 📊 기능 비교

| 항목 | 기존 방식 | 이 도구 |
|------|---------|--------|
| 대시보드 생성 | 도메인마다 수동 생성 | 에픽 키 하나로 자동 생성 |
| 우선순위 확인 | 탭 여러 개 필요 | 한 화면에서 탭 전환 |
| 해결됨 이슈 추적 | 별도 필터링 필요 | 섹션 분리로 즉시 확인 |
| 담당자별 현황 | 조회 불가 | 드로어로 한눈에 확인 |
| 이슈 생성 | 고정값 매번 입력 | 자동 입력 |
| 결과 공유 | 여러 링크 전달 | 파일 하나로 공유 |

## 📦 데모 페이지

`docs/` 폴더의 파일들은 **더미 데이터로 구성된 데모**입니다. 실제 Jira 연동 없이 UI/UX를 미리 체험할 수 있습니다.

| 파일 | 설명 | 용도 |
|------|------|------|
| `demo-before.html` | 대시보드 운영 뷰 | QA 담당자가 실제 사용하는 화면 |
| `demo-after.html` | 스냅샷 공유 뷰 | 내보내기 후 협업 부서에 전달하는 읽기 전용 화면 |

> **주의**: 데모 데이터 특성상 불러오기·이슈생성·내보내기 버튼은 동작하지 않습니다.
